    A class representing a process which is a collection of states. Each process must have at least on entry state and may have at least one exit state.

    All params subject to change, work in progress.

    name

    The name of the state.

    duration

    The duration of the state in sim-units (default: milliseconds)

    on_start_message

    A string message thrown when the entity activates the state (it can only be thrown once, even if the entity enters the state several times, consecutively).

    on_activation_message

    A string message thrown when the entity activates the state, an activation occurrs each time a state is renewed. See TimeFreq

    on_hibernate_message

    A string message thrown when the state is toggled to void - only thrown if the state has a frequency. See TimeFreq

    on_exit_message

    A string message thrown when the entity leaves the state (it can only be thrown once, even if the entity enters the state several times, consecutively).

    onmeta_tags

    A map of key:value pairs that can be searched and indexed while the state is active.

    offmeta_tags

    A map of key:value pairs that can be searched and indexed while the state is in hibernation (void). Only applies if a frequency is set. See TimeFreq

    cost

    The cost of entering a state, by default cost is charged only on state start.

    yields

    The result of the state, what pool modification to perform when the state is exited

    cost_on_activation

    A boolean value, if set to true, the cost will be charged each time a state is activated. Only applies if a frequency is set. See TimeFreq

    yield_on_hibernate

    A boolean value, if set to true, the yields will be awarded each time a state is activated. Only applies if a frequency is set. See TimeFreq

  11.  case class TimeFreq (fmin: Int, fmax: Int, windowmin: Long, windowmax: Long) extends Product with Serializable

    A class representing a time frequency.

    A class representing a time frequency. Usually paired with a state to schedule recurring events. If a state has a time frequency parameter, the entity will be in that state for the Duration or the state a numerber of times according to the fmin and fmax per each random interval of at least windowmin to windowmax. When the entity is not in this state, it will be rendered with a default "void" state.

    All params subject to change, work in progress.

    fmin

    The minimum number of times to occur within the time window.

    fmax

    The max number of times to occur within the time window.

    windowmin

    The minimum length of time window in which the event occurs with some frequency.

    windowmax

    The maximum length of time window in which the event occurs with some frequency.
